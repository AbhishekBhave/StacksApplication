<action>
Install Plaid React Native SDK per current Plaid docs (typical package name `react-native-plaid-link-sdk`). Run `npx pod-install` note in README if iOS native dir exists.

Create `expo/lib/plaid.ts` exporting:
- `fetchLinkToken(): Promise<string>` — calls `supabase.functions.invoke('plaid-link-token', { body: {} })` and returns `data.link_token` (throw on error).

Create `expo/app/(app)/pre-plaid.tsx`:
- Password field + **Continue** + **Cancel**.
- On Continue: `supabase.auth.signInWithPassword({ email: user.email!, password })` using session user email; on success `router.replace` to a small **LinkHost** route or open Link immediately from this screen.
- On Cancel: `router.back()` and **no** Link start.

Wire **Plaid Link** open using SDK: on `onSuccess` send `{ public_token }` to `supabase.functions.invoke('plaid-exchange', …)` then `router.replace('/(app)/transactions')` with query `?justLinked=1` (or equivalent) to trigger success banner.

Update `expo/app/(app)/home.tsx` `onConnectBank`: instead of `connect-bank-placeholder`, navigate to `/(app)/pre-plaid` when email verified.

Add `expo/app/(app)/bank.tsx` — Bank hub shell: shows link status placeholder text, button **Connect bank** (routes to `pre-plaid` when unlinked) and navigation entry to transactions when linked (details in Plan 04).

Register `bank` and `pre-plaid` in `expo/app/(app)/_layout.tsx` stack or tabs per existing router structure.
</action>
