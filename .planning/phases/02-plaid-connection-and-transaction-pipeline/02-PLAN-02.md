# Plan 02 тАФ Supabase Edge Functions for Plaid

<objective>
Implement JWT-gated Edge Functions that mint **link tokens**, exchange **public_token**, run **cursor-based transaction sync**, and **disconnect** (revoke item + wipe user rows per CONTEXT) using **server-only** Plaid secrets.
</objective>

<threat_model>
**ASVS L1 alignment**

| Threat | Mitigation in this plan |
|--------|-------------------------|
| T-2-10 Anonymous link token minting | Each function verifies Supabase JWT (`verify_jwt` in `config.toml` **or** equivalent `Authorization: Bearer` validation); reject missing/invalid JWT. |
| T-2-11 Token exfil in responses | Responses never include `access_token`; `link_token` responses only from `plaid-link-token`. |
| T-2-12 Cross-user item access | Every query scoped by `auth.uid()` from JWT matched to `bank_links.user_id` before reading `plaid_credentials`. |
</threat_model>

<action>
Create four Edge Functions under `supabase/functions/`:

1. **`plaid-link-token`** тАФ `POST`. Read `Authorization` bearer JWT. Resolve `user_id`. Call Plaid `/link/token/create` with `client_user_id = user_id` (string), `products` including `transactions`, `country_codes` including `US`, `language` `en`. Return JSON **only** `{ "link_token": "<token>" }`. Env vars: `PLAID_CLIENT_ID`, `PLAID_SECRET`, `PLAID_ENV`.

2. **`plaid-exchange`** тАФ `POST` body JSON `{ "public_token": string }`. Verify JWT. Exchange via Plaid **`/item/public_token/exchange`** (exact path string). Upsert **one** `bank_links` row with `status = 'active'`, set `plaid_item_id`, `institution_name`, `account_mask` from Plaid metadata when available. Insert `plaid_credentials` row with `access_token`. Enforce at most one active link per user (revoke prior active row or reject тАФ match `02-CONTEXT.md` replace semantics). Trigger initial sync by calling internal sync logic or returning `{ "ok": true }` and letting client call `plaid-sync` immediately after.

3. **`plaid-sync`** тАФ `POST` empty body or `{ "bank_link_id": "uuid" }`. Verify JWT. Load `access_token` for userтАЩs active `bank_link`. Run `/transactions/sync` with cursor from `plaid_sync_state`; upsert into `transactions` using `plaid_transaction_id` + `user_id` uniqueness; update `last_successful_sync_at` on `bank_links`; store new cursor. Filter **out** pending rows **before insert** OR insert all but API to client filters тАФ document choice in function comments.

4. **`plaid-disconnect`** тАФ `POST`. Verify JWT. For userтАЩs active link: call Plaid `/item/remove`, delete `plaid_credentials` row, delete `plaid_sync_state`, `DELETE FROM transactions WHERE bank_link_id = тАж OR user_id = тАж` per CONTEXT, set `bank_links.status = 'revoked'`.

Register all four in `supabase/config.toml` with `verify_jwt = true`.

Use Deno `https://esm.sh/plaid@latest` or official Plaid Node package compatible with Deno per current Supabase Edge guidance тАФ pin a version in import map if required.
</action>
