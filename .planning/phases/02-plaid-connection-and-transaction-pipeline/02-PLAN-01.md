# Plan 01 — Plaid schema, secrets table, and RLS

<objective>
Extend Postgres schema so `bank_links` carries safe display metadata, `transactions` can store Plaid-normalized rows with stable dedupe keys, and **Plaid access tokens** live in a **server-only** table with **RLS default-deny** for `authenticated`.
</objective>

<threat_model>
**ASVS L1 alignment**

| Threat | Mitigation in this plan |
|--------|-------------------------|
| T-2-01 Token theft via RLS misconfig | `plaid_credentials` has **no** `SELECT`/`INSERT`/`UPDATE`/`DELETE` policies for `authenticated` or `anon`; only Edge Functions using **service role** touch rows. |
| T-2-02 Over-broad grants | `REVOKE ALL ON plaid_credentials FROM PUBLIC` (and explicit `authenticated`/`anon` revoke if your template grants them). |
| T-2-03 Orphan secrets on user delete | FK `plaid_credentials.bank_link_id` → `bank_links(id) ON DELETE CASCADE`; `bank_links.user_id` already `ON DELETE CASCADE` from Phase 1. |
</threat_model>

7. Extend `public.transactions`:
   - `ALTER TABLE public.transactions ADD COLUMN IF NOT EXISTS bank_link_id uuid references public.bank_links(id) on delete cascade;`
   - `ALTER TABLE public.transactions ADD COLUMN IF NOT EXISTS plaid_transaction_id text;`
   - `ALTER TABLE public.transactions ADD COLUMN IF NOT EXISTS posted_date date;`
   - `ALTER TABLE public.transactions ADD COLUMN IF NOT EXISTS raw_name text;`
   - `ALTER TABLE public.transactions ADD COLUMN IF NOT EXISTS merchant_name text;`
   - `ALTER TABLE public.transactions ADD COLUMN IF NOT EXISTS pending boolean not null default false;`
   - `ALTER TABLE public.transactions ADD COLUMN IF NOT EXISTS iso_currency_code text default 'USD';`

8. Dedupe index: `CREATE UNIQUE INDEX IF NOT EXISTS transactions_user_plaid_id_key ON public.transactions (user_id, plaid_transaction_id) WHERE plaid_transaction_id IS NOT NULL;`

9. Enable RLS on new tables: `ALTER TABLE public.plaid_credentials ENABLE ROW LEVEL SECURITY;` and `ALTER TABLE public.plaid_sync_state ENABLE ROW LEVEL SECURITY;`

10. **Do not** create any policy granting `authenticated` access to `plaid_credentials` or `plaid_sync_state`. Add comment in SQL: `-- server-only: Edge Functions use service role`.

11. Owner policies for `plaid_sync_state` **read** optional: actually CONTEXT says cursor server-side — **no authenticated SELECT** on `plaid_sync_state` either (same as credentials). If executor prefers client-visible `last_successful_sync_at` only on `bank_links`, keep cursors server-only.

12. `REVOKE ALL ON public.plaid_credentials FROM anon, authenticated;` and `REVOKE ALL ON public.plaid_sync_state FROM anon, authenticated;` (ignore error if role lacks grant — document in migration comment).

No column named `access_token` is added to `bank_links` itself.
