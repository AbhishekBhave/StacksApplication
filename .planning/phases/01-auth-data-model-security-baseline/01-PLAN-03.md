# Plan 03 — Authentication screens and session routing

<objective>
Implement single-screen sign-up/sign-in with toggle, forgot-password flow, and root layout that redirects authenticated users toward onboarding/app per `01-UI-SPEC.md`.
</objective>

<threat_model>
**ASVS L1 alignment**

| Threat | Mitigation |
|--------|------------|
| T-3-01 Weak password UX | Enforce minimum length 8 in client before calling Supabase; surface `AuthApiError` messages without echoing password. |
| T-3-02 Open redirect on recovery | `resetPasswordForEmail` uses fixed `redirectTo` from env `EXPO_PUBLIC_AUTH_REDIRECT` documented in `.env.example` only if Supabase requires it. |
| T-3-03 Anonymous access to app routes | `(app)` group layout checks `supabase.auth.getSession()` and redirects unauthenticated users to `/(auth)`. |
</threat_model>

<action>
Configure `expo-router` route groups:

- `expo/app/(auth)/_layout.tsx` — stack, no auth guard.
- `expo/app/(auth)/index.tsx` — hosts **AuthForm** with `mode: 'signup' | 'signin'` state toggled by UI control (SegmentedButtons or two top tabs).

Implement `expo/components/AuthForm.tsx`:
- Fields: `email`, `password`.
- Sign up calls `supabase.auth.signUp({ email, password })`.
- Sign in calls `supabase.auth.signInWithPassword({ email, password })`.
- Password length validated: `password.length >= 8` before submit.
- On success: `router.replace('/(onboarding)')` for new sign-up path; for sign-in `router.replace('/(app)/home')` — adjust if `onboarding_completed` flag needed (use `profiles` or AsyncStorage key `onboarding_done` string `'true'`).

Add `expo/app/(auth)/forgot-password.tsx` calling `supabase.auth.resetPasswordForEmail(email)` with try/catch and success copy “Check your email”.

Wire root `expo/app/_layout.tsx` or `expo/app/index.tsx` to redirect: if session → `/(app)/home` or onboarding based on flag; else → `/(auth)`.
</action>

</acceptance_criteria>
