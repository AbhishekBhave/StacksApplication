# Plan 04 — Onboarding carousel, home shell, sign-out, and verification gate

<objective>
Deliver onboarding carousel after signup, authenticated home shell with Connect bank CTA gated on `email_confirmed_at`, and sign-out that clears Supabase session plus AsyncStorage keys used by the app (aggressive local clear per CONTEXT).
</objective>

<threat_model>
**ASVS L1 alignment**

| Threat | Mitigation |
|--------|------------|
| T-4-01 Residual PII after logout | On sign-out: `supabase.auth.signOut()` then `AsyncStorage.clear()` **or** multiRemove of all keys prefixed `stacks.` / `@supabase` per implementation — document exact keys cleared in code comment. |
| T-4-02 Bank connect without verification | Before navigating to Plaid (placeholder), check `user.email_confirmed_at`; if falsy, show `EmailVerificationBanner` and do not proceed. |
| T-4-03 Session fixation | Rely on Supabase token refresh; no custom token storage in SecureStore for this phase unless required by Supabase recipe. |
</threat_model>
